# Requerimientos funcionales
1. El sistema debe recopilar los datos de las encuestas periódicamente.
2. El sistema debe analizar variables relacionadas con carga laboral, desempeño, ausentismo y resultados de micro-encuestas internas.
3. El sistema debe identificar patrones que indiquen riesgo potencial de burnout (individual y colectivo).
4. El sistema debe generar alertas tempranas con niveles de riesgo estimado (individual y colectivo).
5. El sistema debe permitir validación humana por parte de Recursos Humanos.
6. El sistema debe ofrecer visualización consolidada para toma de decisiones estratégicas (tendencias mensuales y comparaciones por área).
7. El sistema debe estar desacoplado de la IA.
8. El sistema debe predecir con exactitud.
9. El sistema debe implementar intervenciones preventivas oportunas (ajuste de carga, acompañamiento y redistribución de tareas).
# Requerimientos no funcionales
1. El sistema debe ser escalable y técnicamente sostenible.
2. El sistema debe soportar por lo menos 225 peticiones al tiempo.
3. El sistema debe tener un 90% de uptime. 
4. El sistema debe tener un MTTR (Mean Time To Recover) de $\approx 15$ mins.
5. El sistema debe ser asíncrono.
6. El sistema debe contar con una documentación técnica amplia.
7. El sistema debe cumplir con los estándares de protección de datos y privacidad laboral.
8. El sistema debe tener métricas de evaluación de desempeño pertinentes (accuracy, precision, recall).
9. El sistema debe ser modular y escalable.
10. El sistema de operar en un entorno de bajo costo.
